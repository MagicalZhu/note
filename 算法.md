

## 时间复杂度
> 时间复杂度指输入数据大小为 `N` 时，算法运行所需花费的时间


有两个需要注意的是:
1. 统计的是算法的`计算操作数量`，而不是`运行的绝对时间`, 计算操作数量与运行的绝对时间呈`正相关`,但是并不相等
2. 体现的是计算操作 随着输入数据N变化后的情况,所以假设某个算法执行1次,或者执行100次,他们的时间复杂度是O(1)。执行N次和100N次,那么时间复杂度都是O(n)

时间复杂度通常有以下的种类:
`O(1)< O(logN) < O(N) < O(NlogN) < O(N^2^) < O(2^N) < O(N!)`

### 常数O(1)
![](https://pic.leetcode-cn.com/1623779241-lViysV-Picture2.png)

**O(1)的示例1: 输入数据N参与计算**
```java
int algorithm(int N) {
    int a = 1;
    int b = 2;
    // 这里a,b,N无论取多大的值参与计算,都只会运行一次,直接返回结果1
    int x = a * b + N;
    return 1;
}
```

**O(1)的示例2: 输入数据N没有参与计算**
```java
// 这个函数无论输入数据N多大,都不会影响计算数量
int algorithm(int N) {
    int count = 0;
    int a = 10000;
    for (int i = 0; i < a; i++) {
        count++;
    }
    return count;
}
```
